# Task Rules

Each task is one markdown file. Work starts in `tasks/ready/` and ends in `tasks/review/` with a completion note.

Task states:
- `tasks/inbox/`: untriaged ideas.
- `tasks/ready/`: executable tasks.
- `tasks/in_progress/`: currently owned work.
- `tasks/review/`: completed work awaiting review.
- `tasks/done/`: accepted work.
- `tasks/blocked/`: work that cannot proceed.

Execution rules:
- Read the task completely.
- Read only relevant docs and files.
- Change only requested paths.
- Produce the stated output.
- Verify acceptance criteria.
- Move the task forward exactly one state unless blocked.
